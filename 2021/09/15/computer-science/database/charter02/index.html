# 2.1 关系数据结构及形式化定义

# 2.1.1 关系

  • 关系 —— 单一的数据结构
  • 二维表 —— 关系模型中数据的逻辑结构
  • 域（Domain） 是一组具有相同数据类型的值的集合。如整数、实数、指定长度的字符串等
  • 笛卡尔积（Cartesian Product） ：域上的集合运算
    • 元组
    • 分量
    • 基数：一个域允许的不同取值个数。
  • 关系（Relation）
  • 码
    • 候选码：若关系中的某一属性组的值能唯一地标识一个元组，则称该属性组为候选码。极端情况下，关系模式的所有属性组是这个关系模式的候选码，称为全码（All-key）。
    • 主码：若有多个候选码，则选定一个作为主码。
      • 候选码的诸属性称为主属性。不在候选码中的属性叫做 非主属性 或 非码属性 。
  • 三类关系：
    • 基本关系：实际存在的表。
      • 列是 同质的（homogeneous） , 即每一列的分量的数据类型相同。
      • 不同的列可以出自同一个域，属性名要不一样。
      • 列的顺序无所谓。
      • 行的顺序无所谓。
      • 任意两个元组的候选码不能取相同的值。
      • 分量必须取原子值，不可再分。
    • 查询表：查询结果对应的表。
    • 视图表：由基本表或其他视图导出的表，是虚表。

# 2.1.2 关系模式

  • 关系的描述称为关系模式（relation schema）。
  • 静态的、稳定的。

# 2.1.3 关系数据库

  • 关系数据库的型也称为关系数据库模式，是对关系数据库的描述。
  • 关系数据库的值是这些关系模式在某一时刻对应的关系的集合，通常就成为关系数据库。

# 2.2 关系操作

# 2.2.1 基本的关系操作

  1. 查询操作（query）
    • 选择（select）
    • 投影（project）
    • 连接（join）
    • 除（divide）
    • 并（union）
    • 差（except）
    • 交（intersection）
  2. 插入（insert）、删除（delete）、修改（update）
  • 关系操作的特点是集合操作方式，操作的对象和结果都是集合

# 2.2.2 关系数据语言的分类

  • 关系数据语言
    • 关系代数语言（如 ISBL）
    • 关系演算语言
      • 元组关系演算语言（如 ALPHA、QUEL）
      • 域关系演算语言（如 QBE）
    • 具有关系代数和关系演算双重特点的语言（如 SQL）

# 2.3 关系的完整性

# 2.3.1 实体完整性

  • 实体完整性规则：主属性不能为空值。

# 2.3.2 参照完整性

  • 设 F 是关系 R 的属性，但不是关系 R 的码，K 是关系 S 的主码，若 F 与 K 对应，则 F 是 R 的 外码（foreign key） ,R 为 参照关系（referencing relation） ,S 为 被参照关系（referenced relation） 或 目标关系（target relation） 。
  • 参照完整性规则：若 F 是关系 S 的外码，则对于 R 中每个元组在 F 上的值必须：
    • 要么为空值
    • 要么等于 S 中某个元组的主码值

# 2.3.3 用户定义的完整性

  • 用户定义的完整性就是针对某一具体关系数据库的约束条件，反映某一具体应用所涉及的数据必须满足的语义要求。

# 2.4 关系代数

  • 关系代数是一种抽象的查询语言，用于对关系的运算表达查询。

# 2.4.1 传统的集合运算

  • 二目运算，包括并，差，交，笛卡儿积 4 种运算
  • 设关系 R、S 具有相同的属性，相应属性值取自同一个域，t 为元组。
    1. 并（union）: R∪S={t|t∈R∨t∈S} ,t 属于 R 或属于 S。
    2. 差（except）: R-S={t|t∈R∧t∉S} ,t 属于 R 且不属于 S。
    3. 交（intersection）: R∩S={t|t∈R∧t∈S} ,t 属于 R 且属于 S。 R∩S=R-(R-S)
    4. 笛卡儿积:m 和 n 分别为 R 和 S 的列，k1 和 k2 分别为 R 和 S 的元组，则 R×S 有（m+n）列，有（k1*k2）行。
