# 第一章 概述

# 1.1 网络安全的定义

计算机系统安全

  • 计算机的硬件、软件和数据得到保护，不因偶然和恶意的原因而遭到破坏、更改和泄露，保障系统连续、正常地运行。
    网络安全
  • 在通信网络 (通信子网 no 资源子网) 中对信息载体和信息的传输、处理、存储、访问提供安全保护，以防数据和信息内容遭到破坏、篡改和泄露，或网络服务中断、或拒绝服务、或被非授权使用。
  • 信息载体：能携带信息的媒介，是信息赖以附载的物质基础，即用于记录、传输、积累和保存信息的实体。信息载体包括：
    • 无形载体：以能源和介质为特征，运用声波、光波、电波传递信息。
    • 有形载体：以实物形态记录为特征，运用纸张、胶卷、胶片、磁带、磁盘传递和贮存信息。
      信息安全
  • 信息在整个网络 (通信子网 + 资源子网) 范围内的采集、传输 (网络安全)、存储、处理 (计算机系统安全)、访问过程中不会被破坏、窃取和篡改，并且网络服务和网络资源能够连续、正常地被合法用户访问。
    网络安全防护的五大目标
  1. 保密性
  • 保障手段：加密技术
  • 机密性：敏感 (隐私或机密) 的信息不会被泄露给未经授权的个体或团体。
  • 隐私性：信息的所有者 (个体或团体) 仅可以控制和操作 (增删改查) 与之相关的信息。
  1. 完整性
  • 保障手段：报文摘要、基于加密技术的数字签名等技术。
  • 信息内容的完整性：不被传错、偶然 (非故意) 或蓄意 (故意) 地篡改 (插入、删除、修改、伪造、乱序、重放)。
  • 信息发送方身份 (信息来源) 的真实性：非冒充。
  1. 不可抵赖性
  • 保障手段：基于加密技术的身份认证、数字签名等技术
  • 通信的各方都不能否认曾经完成的操作
  1. 可控性
  • 保障手段：查杀恶意代码、抵御拒绝服务攻击等防御技术
  • 信息能够被授权实体连续、正常地访问 (操作)
  1. 可控性
  • 障手段：基于加密技术的访问控制等技术
  • 仅允许被授权的实体在操作权限范围内访问信息资源

# 1.2 网络系统面临的安全威胁

  1. 恶意代码
  • 一种程序代码：经过存储介质和网络进行传播，从一台计算机到另一台计算机，未经授权、认证，破坏计算机系统的完整性
  • 计算机病毒 (Computer Virus): 通过移动存储或网络自我复制，对系统造成巨大破坏.
  • 蠕虫 (Worms): 自动寻找网络漏洞，自我复制传播，连接网络，消耗主机和网络资源，控制主机
  • 特洛伊木马 (Trojan Horse): 与远程主机建立连接，使得本地主机对远程主机进行控制、破坏、窃取信息
  • 逻辑炸弹 (Logic Bombs)：在特定条件满足时，对潜入的目标设备实施毁灭性破坏
  • 系统后门 (Backdoor): 绕过安全控制措施，获取对程序或系统的高度访问权
  • Rootkit：给自身文件、进程加持长久的操作系统根权限，目的是控制系统、窃取信息
  • 恶意脚本 (Malicious Scripts)：直接危害用户，或者损害系统功能、间接地危害用户
  1. 远程入侵
  • 有意违反系统安全策略、侵犯安全服务的行为
  • 远程攻击
    • 非法接入：非授权人员连接到网络系统内部并获得访问系统内部资源的途径
    • 非法访问：非授权用户通过远程登录或黑客工具远程访问主机的信息资源
  • 非授权人员 (入侵者): 没有事先获得信息资源操作权限的人及其使用的设备、发起的进程
    • 黑客 (Hacker)：精通网络、外设以及计算机软硬件的程序员，为研究安全技术而研发攻击工具、发动攻击.
      黑客精神：
    • Free (自由、免费)：将自己的心得和编写的工具和其他人无偿地分享
    • 探索与创新：喜欢探索软件程序奥秘
    • 反传统 (已存在的事物)：找出系统漏洞，研发相关的攻击手段，利用该漏洞进行攻击
    • 合作：需要数人或更多人的通力协作才能完成任务。
    • 骇客 (Cracker)：具有一定水平的程序员，为了获取私利而恶意地破解或破坏某个文件、进程、损害网络
  1. 拒绝服务攻击
  • 定义:
    • 让被攻击的主机 (目标主机) 或其服务系统 (如 IIS) 停止提供正常服务或资源访问
    • 资源：磁盘空间、内存、进程、网络带宽、......
  • 主要攻击方式之一：对网络带宽进行的消耗性攻击
    • 向服务器发送大量 IP 分组，导致正常用户请求服务的分组无法到达该服务器
  • 主要攻击方式之二：利用系统漏洞，访问系统，使得系统崩溃
    • 利用 C 程序中存在的缓冲区溢出漏洞进行攻击，发送精心编写的二进制代码，导致程序崩溃，系统停止服务
  1. 身份假冒
  • IP 地址假冒
    • 用合法用户的或不存在的 IP 地址，作为攻击者自己发送的 IP 分组的源 IP 地址
    • IP 路由协议的漏洞：不检查 IP 分组的源 IP 地址
  • 用户身份假冒
    • 身份信息：一组特定的数据（比如：用户名 tangbohu + 密码 tbh9257 ）
    • 利用社会工程学方法或网络监听的方式窃取这些特定数据
    • 利用这些数据假冒合法用户，欺骗远程系统
  1. 信息窃取和篡改
  • 主动攻击：改变信息资源
    • 重放 (replay)：在接收方收到信息之后，向接收方发送接收过的信息，对接收方的身份认证过程进行欺骗
    • 篡改： 非授权地修改存放的信息资源，或窃取传输中的信息、进行修改、延迟或重排、再发给接收方
    • 冒充：用窃取到的合法用户身份信息，通过身份验证，从而冒充该合法用户访问信息资源
    • 伪造：冒充发送方的身份，制造虚假信息，并发给接收方
    • 阻断：恶意中断通信双方的信息传输过程，是针对可用性的一种攻击
  • 被动攻击：不改变信息资源
    • 在通信双方的物理线路上，安装信号接收装置，窃听通信内容
    • 使用流量分析技术，观察信息的频率和长度，推测通信双方的位置和身份

# 1.3 网络安全的研究内容

  1. 网络攻击技术
  • 网络攻击的定义
    • 对网络安全防护的目标对象产生危害的行为
    • 4 种类型：非法访问、信息泄露、完整性破坏、拒绝服务攻击
  • 基本特征
    • 使用一定的攻击工具对目标网络系统进行攻击访问
    • 收获到某种程度的攻击效果，实现了攻击者的攻击意图
  • 攻击方式
    • 读取攻击：用于侦查 (监听和扫描) 要攻击的目标，识别目标主机运行的网络服务以及可能的漏洞
    • 操作攻击：以篡改数据为手段，攻击并取得程序的控制权，如缓冲区溢出攻击
    • 欺骗攻击：将自身伪装成其他用户实施攻击行为，如网络钓鱼
    • 泛洪攻击：让远程主机无法承受巨大的流量而瘫痪，如 TCP SYN Flood 和 DDoS 攻击
    • 重定向攻击：将发往接收方主机的信息全部重定向到攻击者指定的目标主机上，如 ARP 重定向
    • Rootkits 技术：用于隐藏自身及指定文件、进程和链接的恶意软件工具集，目的是控制系统、窃取敏感信息
  • 几种常用的攻击手段
    • 网络监听：只要攻击者获取数据通信的传输路径即可轻易实现监听
    • 篡改数据：攻击者迅速完成对数据的截获、修改、发送，使得数据收发双方很难察觉
    • 网络欺骗：常见的欺骗攻击主要有 IP 欺骗、ARP 欺骗、DNS 欺骗
    • 弱口令攻击： 通过各种方式成功获取或破解合法用户的简单口令（密码）
    • 拒绝服务：使攻击的目标系统停止工作或耗尽目标主机的网络带宽，以致无法响应其他主机的服务请求
    • 漏洞破解：利用系统漏洞实施攻击，获取系统访问权限
    • 木马攻击：植入恶意代码或链接，诱使用户查看或点击，然后自动下载木马程序到到目标用户主机中运行
  • 攻击流程
    1. 信息收集：通过各种方式获取目标主机或目标网络的若干信息，属于攻击前的准备阶段
      • 收集的信息
        • 网络接入方式：拨号接入、无线局域网接入、以太网接入、VPN 远程接入等
        • 目标网络信息：域名范围、IP 地址范围、地理位置等
        • 网络拓扑结构：网络拓扑结构、网络协议、交换设备类型、设备生产厂家等
        • 网络用户信息：邮件地址范围、用户账号密码等
      • 收集方式
        • 使用主流的搜索引擎。如 Google、必应、百度等
        • 使用 whois 服务器查询主机的具体域名和地理信息
        • 使用 netdiscover 等工具查询主机的 IP 地址范围
        • 使用 DNSmap 、 DNS walk 、 dig 等工具查询域名、空间所在服务器
        • 使用社会工程学手段获得目标人的信息（姓名，电话，家庭住址，孩子姓名等）
    2. 网络隐身：在网络中隐藏自己真实的 IP 地址，使受害者无法反向追踪到攻击者
      • IP 假冒或 IP 盗用：TCP/IP 协议不检查 IP 分组的源 IP 地址，修改 IP 地址绕过访问控制中的黑名单
      • MAC 地址盗用：修改自身主机的 MAC 地址为允许访问的 MAC 地址
      • 代理隐藏：通过免费代理进行信息收集，甚至通过多个代理级联
      • 冒充真实用户：监听或破解用户的账号和口令，然后冒充该用户
      • 僵尸机器：作为直接攻击者、挡箭牌（僵尸主机）入侵其他主机，进行攻击
    3. 端口和漏洞扫描
    • 端口扫描
      • 检测有关端口是打开还是关闭
      • 判定目标端口运行的服务类型和版本信息
      • 识别不同操作系统的类型和版本
    • 漏洞扫描
      • 通用的漏洞扫描工具，如 Nessus 、其开源版本 Openvas 和国产版本 X-scan
      • 专用的漏洞扫描工具，如 WEB 应用程序的漏洞扫描工具 Nikto 、 Golismero 数据库漏洞扫描工具 NGS SQuirrel
    1. 实施攻击
    • 获取访问权限
      • 利用远程漏洞进行远程入侵，获得目标系统的一个普通用户权限
    • 提升访问权限
      • 配合本地漏洞把获得的权限进行提升，提升为系统管理员的最高权限
      • 暴力破解管理员口令、检测系统配置错误、网络监听或设置钓鱼木马
    • 实施攻击过程
      • 注：拒绝服务攻击的危害极大，目前还没有防御 DoS、DDoS 的好办法
        网络攻击的发展方向：
    • 平台化：利用平台提供的现有功能模块，开发新的攻击工具
    • 集成化：新的攻击工具可以嵌入其他类型的攻击工具
      1. 设置后门
      • 开放不安全的服务端口：比如，TCP 的 135、139 端口， UDP 的 445 端口，流行病毒的 TCP 2745、6129 端口，远程服务访问端口 3389
      • 修改系统配置：比如，白名单，入站规则，密码规则
      • 安装网络嗅探器：比如， sniffer
      • 建立隐藏通道：比如，反弹 shell 、端口转发、内网穿透
      • 创建具有 root 权限的虚假用户账号：比如， administrater ， sysbakor
      • 安装批处理程序并设置激活条件：比如， addcounts.bat weekx-3:00
      • 安装远程控制木马：比如，上兴远程 or 灰鸽子
      • 使用木马程序替换系统程序：比如，将 backdoor-factory 改名、替换某个常启动的系统程序
    1. 清除痕迹：成功获得访问权后，必须清除所有痕迹，防止被管理员发现
      • 清除登录日志和其它有关记录
        • 隐藏上传的文件
        • 修改日志文件中的审计信息
        • 修改系统时间造成日志文件数据紊乱
        • 删除或停止审计服务进程
        • 干扰入侵检测系统正常运行
        • 修改完整性检测数据
        • 使用 Rootkits 工具
      • Windows 的系统日志 “%systemroot%\system32\config”
  1. 网络安全防护体系
  • 大量的现有攻击技术和频出的新型攻击技术，给信息资源及其所在的网络带来了诸多网络安全问题
  • 任何一种单一安全防护技术都无法有效解决所有的网络安全问题
  • 最新的思路：
    • 必须在网络的每一层次增加相应的安全防护功能
    • 各层的安全防护功能联动 (协调互助)，构成有机的网络安全防护体系
    • 无法构建一个适用于所有网络应用环境的网络安全防护体系
      • 研究、构造网络安全防护体系，必须与具体应用环境的安全需求相结合
    1. 物理层安全：物理环境的安全性，包括通信线路的安全、物理设备的安全和机房安全
      • 防盗：偷窃行为所造成的损失可能远远超过主机本身的价值
      • 防火：
      • 电气设备和线路因为短路、过载 (电压，电流，工作时间) 等原因引起电打火而导致火灾
      • 由于操作人员不慎，使存在易燃物的机房着火
      • 因外部房间或其他建筑物起火而蔓延到机房而引起火灾
      • 防静电：计算机显示器、主板、线路、接口等容易积累电荷，从而产生很强的静电
      • 防雷击：雷击可能使大规模集成电器 (部件多 or 含高集成芯片) 损坏，这种损坏往往不易察觉
      • 防电磁泄漏：电磁发射包括辐射发射和传导发射
    2. 系统层安全：操作系统的安全性是整个网络与计算机系统的安全基础
    • 及时修复系统漏洞
      • 系统漏洞的来源：操作系统在开发过程中存在的技术缺陷或程序错误。导致其他用户非法访问或利用恶意代码进行攻击
      • 应对方法：关注操作系统供应商的安全公告，第一时间打上相应的官方补丁
    • 防止系统的安全配置错误
      • 必须根据具体应用环境的安全需求，对系统提供的服务进行安全配置
    • 错误的配置导致安全风险
      • 手工或使用安全配置检查工具，检查配置是否与预定义的安全策略保持一致
    • 防止病毒对系统的威胁
      • 及时防毒：采取相应的系统安全措施预防病毒侵入计算机
      • 及时查毒：对于确定的环境准确地识别病毒名称
      • 及时杀毒：根据不同类型的病毒，恢复感染对象的正常状态
    1. 网络层安全
      • 身份认证
        • 确认操作者的身份是否合法
        • 附带工作：确定操作者是否拥有对某种信息资源的访问权限
      • 访问控制
        • 防止非法主体进入受保护的信息资源
        • 允许合法用户访问受保护的信息资源
        • 防止合法用户越权访问受保护的信息资源
      • 数据传输的保密性和完整性
        • 信息在传输过程中不泄露给非法的第三方
        • 保护信息内容的完整性：不被传错、篡改
      • 路由系统安全
        • 路由器操作系统的安全：给路由器打上漏洞补丁，根据安全策略检测路由的安全配置是否正确
        • 路由信息传输的安全：保证路由信息的保密性和完整性，防止攻击者发送和传播伪造的路由信息
      • 入侵检测
        • 从网络或系统安全日志等渠道收集和分析与网络行为有关的信息，从中检查是否存在违反安全策略的行为和被攻击的迹象，若存在则采取相应的措施 (阻断 / 取证，起诉)
        • 提供对内部攻击、外部攻击和误操作的实时保护
      • 防病毒技术 (这里是指广义的病毒：恶意代码)
        • 防病毒工具：个人版杀毒软件 / 防毒墙 + 企业版工具 (n x 客户端软件 + 服务器端)
        • 通过大量客户端监测网络中软件行为的异常行为，获取病毒的最新信息
        • 推送到服务端进行自动分析和处理，再把解决方案分发到每一个客户端
        • 依靠庞大的网络服务器，实时进行采集、分析以及处理
    2. 应用层安全
      • WEB 安全
        • 互联网中的 WEB 服务器使用 HTTP 协议传输数据，该协议没有考虑安全因素
          • 账号和密码信息未经加密即在客户和服务器之间传输
          • HTTP 是无状态的协议，基于 WEB 的身份冒充非常容易
          • WEB 应用程序可能存在诸多安全漏洞
      • DNS 安全
        • DNS 欺骗：伪造假的 DNS 应答给 DNS 查询方 (如：浏览器)，将用户引导到错误的站点
        • 防御 DoS 攻击：启用备份域名服务器、最小权限原则、限制区域传输、提供最少服务
      • 邮件系统安全
        • 直接攻击：窃取邮箱密码、截获邮件内容、伪造邮件内容、发送垃圾邮件 (消耗邮箱存储空间和用户时间)
        • 间接攻击：将恶意代码放在邮件附件中或者伪造成网页或链接，诱骗用户点击
    3. 管理层安全
    • 法律法规：不健全、宣传不足、执行不彻底
    • 网络管理人员：技术能力不足，安全意识不到位，防护有漏洞或薄弱环节，操作不遵守规范，工具不安全
  1. 网络防御技术
  2. 信息加密：是绝大多数其他网络安全技术的核心和基石
    • 数据加密 / 数据解密
    • 消息摘要
    • 数字签名
    • 密钥交换
  3. 访问控制（广义）：基于身份认证、资源访问授权、验证用户的访问权（允许 or 拒绝）
  • 身份认证：用户要向系统证明 TA 就是 TA 所声称的合法用户
    • 身份识别：获取并辨认出访问者的身份信息，如：用户名 + 密码，指纹
    • 身份验证：核对身份信息，确认身份是否合法
      • 单机环境认证：防止非法用户登录主机、窃取 or 破坏信息资源
        认证方式：基于口令、智能卡、生物特征 (指纹 / 声音 / 脸型 / 虹膜) 的认证
      • 网络环境认证：防止网络中传输的身份信息被攻击者窃取和重放
        认证方式：挑战握手认证、Kerberos 认证协议、TLS 认证协议
  • 访问控制 (狭义)：主流的方法：自主访问控制、强制访问控制、基于角色的访问控制
  • 访问控制系统
    • 主体：主动发出操作请求的一方 (用户 or 进程)
    • 客体：主体试图访问的信息资源
    • 安全访问策略：一套规则，确定主体是否对客体拥有访问能力
  1. 防火墙：在单台主机上 or 不同网络之间，对网络流量或访问行为实施访问控制的一套组件
  • 基本条件
    • 内网和外网之间的所有数据流必须经过防火墙
    • 只有符合安全策略的数据流才能通过防火墙
    • 自身具有高靠性（工作状态稳定，不轻易罢工）
- 主要功能
    - ==服务控制==：只允许子网间相互交换与特定服务有关的信息
    - ==方向控制==：只允许由某个特定子网or终端发出的信息通过
    - ==用户控制==：为每个用户设定访问权限，对访问资源的用户进行认证
    - ==行为控制==：对访问资源的操作行为进行控制(允许or拒绝)和记录
  1. 入侵防御（IPS）
  • 入侵
    • 未经授权，蓄意访问、篡改数据，使网络系统不可使用的行为
- 入侵防御
    - 对行为特征、安全日志、审计数据或其他网络上可以获得的信息进行检测
    - 检测到对系统的入侵或入侵企图，并及时阻止入侵
  1. 恶意代码规范：在一定环境下可以独立执行的完整程序或嵌入到其他程序中的代码段
  • 不具有复制能力的依附性恶意代码
    • 木马、逻辑炸弹、后门
- 不具有复制能力的独立性恶意代码
   - ++木马、后门、Rootkits++{.dot}
- 具有自我复制能力的依附性恶意代码
   - ++病毒++{.dot}
- 具有自我复制能力的独立性恶意代码
   - ++蠕虫、恶意脚本++{.dot}
  • 防范方法
    • 基于特征的扫描技术
      • 反病毒引擎最常用的技术
      • 将恶意代码的特征存入特征库，在扫描时根据特征库行查找、匹配、判断
    • 校验和法
      • 在系统未被感染前，对待检测的正常文件计算出其校验和
      • 周期性地计算出其校验和，与正常校验和比对，侧判断文件的改变情况
    • 沙箱技术
      • 根据程序需要的资源和拥有的运行权限，建立程序的运行沙箱
      • 每个程序都运行在自己的沙箱中，无法影响其它程序的运行
      • 在沙箱中可以安全地检测是否存在恶意代码、分析恶意代码的行为
    • 基于蜜罐（影子，替身）的检测技术
      • 蜜罐是一种虚拟系统，伪装成运行许多服务的主机以吸引攻击
      • 安装强大的监测系统，用于监测恶意代码的攻击过程用以研究攻击技术或取证
  1. 安全审计与查证：集实时收集和监控网络各组件的安全状态和安全事件，以便中报警、分析和处理
  • 主要作用
    • 对潜在攻击者起到震慑或警告作用
    • 对已发生的系统破坏行为提供有效证据
    • 提供有价值的日志，帮助管理员发现系统入侵行为或潜在的系统漏洞
    • 提供系统运行的统计日志，发现系统性能的脆弱点
- 主要内容
    - 网络设备审计：【审计的对象】路由器等设备的配置信息、用户权限、链路带宽
    - 日志文件审计：【日志的内容】记录了系统运行情况，通过它可以发现攻击痕迹
    - 安全威胁审计：【旨在发现】未经授权的资源访问、数据修改和操作
  1. 网络安全应用
  • 网络通信协议：完成数据传输任务的规则。设计这种软件时很少考虑数据传输的安全保障功能。如： PPP，CSMA/CD，IP，ARP，TCP，UDP，http，DNS
  • 网络安全协议：弥补通信协议的安全缺陷，这种规则保护传输的信息。如： 802.1x， 802.11i，IPSec，SSL，VPN，https，SET
    1. 802.1X: 在有线网络中的链路层提供认证服务，基于端口的接入控制协议
    • 对交换机端口加以控制以实现用户级的接入控制
    • 通过扩展认证协议 EAP 完成对接入用户的认证
    • 端口未授权时，只有 EAP 报文和广播报文可以通过端口转发
    • 授权后，认证方才会对请求方的数据提供正常的传输服务
    1. 802.11i: 在无线局域网中的链路层提供认证、数据完整性、数据保密性和密钥管理服务
    • 主要功能
      • 认证：定义用户和认证服务器之间交换的认证协议，并产生临时密钥
      • 访问控制：强制使用认证功能，帮助密钥交换，能够基于认证协议工作
      • 完整性加密：MAC 层数据与无线信号的完整性字段一起加密
    1. IPSec: 一种端到端的确保 IP 层通信安全的机制，是一组协议集
    • 鉴别头部协议 AH
      • 数据完整性验证、数据源身份认证、防重放攻击
    • 封装安全载荷协议 ESP
      • 增加数据加密服务
    • 密钥交换协议 IKE
      • 密钥管理，协商加密参数
    • 安全关联 SA
      • 存储 IKE 的协商参数
    • 解释域 DOI
      • 统一分配标识符，解释参数
    1. SSL: 在应用层协议和传输层 TCP 协议之间保护数据的安全性，提供保密性、完整性、服务器认证和可选的客户机认证
    • SSL 握手协议
      • 预先建立一个连接双方的安全通道
      • 使用公钥算法在客户端验证服务器端的身份
      • 传递客户端产生的对称会话密钥到服务器端
    • SSL 记录协议
      • 封装更高层的协议，执行数据的安全传输
      • 用会话密钥来加密、解密数据
    1. 虚拟专用网 VPN: 建立在公用网上，由某个组织或某些用户专用的通信网络
    • 专用性
      • VPN 之外的用户无法访问 VPN 内传输的信息
      • VPN 内部用户之间可以实现安全通信
    • 解决方案
      • 内联网 VPN、外联网 VPN、远程接入 VPN
    • 关键技术
      • 隧道技术 (Tunnel)：将待传输的信息经过加密和协议封装后，嵌入另一种协议的数据报文中
      • 加解密技术：隧道传输的信息必须加密、解密
      • 密钥管理技术：采用密钥交换协议动态分发密钥
      • 身份认证技术：建立隧道之前必须确认用户身份
    1. SET: 在应用层，目前保证信用卡数据安全的最安全协议
    • 发卡机构
      • 向持卡人发卡并开设账户，如：建设银行
      • 向商家支付持卡人消费的金额
    • 商家结算机构
      • 为商家建立账户，如：支付宝、微信
      • 验证持卡人支付卡的有效性
      • 和发卡机构协调，完成货款支付
    • 支付网关
      • 实现互联网和支付网络之间的互联
      • 实现 SET 消息和电子转账消息之间的相互转换
    • 认证中心链
      • 持卡人、商家、支付网关、商家结算 - 机构、发卡机构都信任的证书签发机构
  1. 密码技术应用
    网络安全的核心建立在密码学的理论和技术之上
  • 密码技术
    • 密码算法设计、密码分析、安全协议、身份认证、数字签名、密钥管理
  • 密码技术的用途 (支撑 NII 的 5 大安全防护目标)
    • 为信息资源提供：机密性、完整性、不可抵赖性、可用性、可控性
  1. 密码学与密码体制
  • 密码编码：研究如何构造一个符合安全要求的密码系统
    - 密码分析：试图破译加密算法和密钥
    - 发展阶段：
    • 古典密码学：远古～1949，数据安全基于算法的保密
    • 对称密码学：1949~1975，数据安全基于密钥的保密
    • 公钥密码学：1976~，通信双方不需要传输全部密钥即可实现保密通信
- 2种密码体制(加密、解密的方式)：
    - ++对称加密++{.dot}
    加密和解密使用相同的密钥，如DES、AES算法
    - ++非对称加密(公钥加密)++{.dot}
        - 加密和解密密钥使用不同的密钥:即配对的公钥和私钥
        - 公钥公开，私钥保密，如D-H、RSA和椭圆曲线算法
- 保密通信系统模型
    - M：明文消息空间，是所有明文（m1，m2，……）的有限集合
    - C：密文消息空间，是所有密文（c1，c2，……）的有限集合
    - K1,K2：所有加密、解密密钥分别构成的有限集合
    - Ek1：加密变换算法
    - Dk2：解密变换算法
![保密通信系统模型](https://z3.ax1x.com/2021/09/08/hbStaV.png)
  1. 数据加密 / 解密技术
  • 加密 / 解密算法
    • 对称算法
      • DES ：Data Encryption Standard，数据加密标准
      • IDEA ：International Data - Encryption Algorithm，国际数据加密算法
      • AES ： Advanced Encryption Standard，AES），高级加密标准
    • 非对称 (公钥) 算法
      • RSA ：三位数学家 Rivest、Shamir 和 Adleman 名字的缩写
      • 椭圆曲线 ：利用基于 MD5 或 SHA-1 摘要函数的散列算法
        注意：Diffie-Hellman 是 Whitefield Diffie 和 Martin Hellman 研发的一种秘钥交换算法，用于建立秘钥，不是加密方法
- 网络加密方式
    - 链路加密
        - 每条链路两端共享一个密钥，在不同的链路上采用不同的密钥
        - 缺点：当信道上链路很多时，需要提供很多密钥
    - 节点加密
        - 中间节点(路由器，三层交换机)装有加解密保护装置，由它完成不同链路的密钥切换工作
        - 缺点：需要修改网络交换节点，增加保护装置
    - 端到端加密
        - 通信线路两端进行加密
        - 缺点：在传输层或应用层上实现，不加密更低层的信息，易受流量分析攻击
-  密码分析
    - 定义：在不知道解密密钥的情况下，对加密信息进行解密：
        - 【合法用户】忘记：密码分析
        - 【攻击者】不知道：破坏性密码分析，破解
    - 使用的技术：
        - 唯密文攻击
        - 已知明文攻击
        - 选择明文攻击
        - 选择密文攻击
        - 自适应选择明文攻击
  1. 认证技术：用于验证所传输的数据的完整性
  • 消息认证（方法：消息认证码、安全散列函数、数字签名）
    • 用于保证信息的完整性和不可否认性
    • 验证消息的发送方身份是否真实
    • 验证消息的发送顺序和及时性
- 身份认证（方法：身份信息的提取与识别，身份信息的查找与比对）
    - 身份识别
    - 身份验证
  1. 公钥基础设施（PKI）: 利用公钥加密，为保障信息安全
  • PKI 策略：定义密码系统使用的处理方法和原则
    - 认证机构 (CA)：管理证书
    注册机构 (RA)：提供用户和 CA 之间的接口
    - 证书 / CRL 发布系统：提供证书的在线浏览、查询和用户登记功能。
    - PKI 应用接口：为外界提供安全服务的入口
